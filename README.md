
No desenvolvimento do projeto Gennesis, a escolha estratégica foi codificar utilizando o Vite. Esta decisão foi pautada na necessidade de entregar uma plataforma de alta performance, focada na experiência do usuário e na agilidade do desenvolvimento.

Por que Vite?
HMR (Hot Module Replacement) Ultrarrápido: Ciclos de feedback instantâneos durante o desenvolvimento, permitindo ajustes em tempo real sem perda de estado.

Build de Produção Otimizado: Utilização do Rollup para gerar bundles extremamente leves, garantindo que o dicionário carregue com velocidade máxima, mesmo em conexões instáveis.

Arquitetura Moderna: Suporte nativo a ES Modules, eliminando o overhead de empacotamento desnecessário durante o desenvolvimento.

Gestão Inteligente de Assets: Processamento eficiente de recursos multimídia, essencial para a proposta enciclopédica do projeto.
